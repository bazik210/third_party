libfios
----------------------------------------------------------------------
- 以下の不具合を修正しました。

  - --xml オプションを指定時、PSARC ツールがアーカイブファイルを作成できない
  （技術情報ID:200909-09 https://ps3.scedev.net/technotes/view/953）

  - 現在、PSARCツールは、PSARC-MSELFアーカイブに含まれる暗号化されたファ
    イルを、適切なオフセットでアラインしていません。これにより、多少読み
    込みパフォーマンスが低下するおそれがあります。将来のPSARCツールでは、
    この問題は修正される予定です。

  - (B#72975)
    PSARC-MSELFアーカイブファイルに含まれるSDATAファイルのサイズを 
    cell::fios::scheduler::getFileSize()もしくは 
    cell::fios::scheduler::getFileSizeSync()により取得する際、属性に
    cell::fios::kOPF_SDATA を指定すると、そのSDATAファイルの元のサイズが
    返るべきですが、SDATAファイルのサイズが返っていました。

  - (B#76309)
    cell::fios::overlayは、複数のファイルが指定したパスに適合する場合、
    誤ったファイルにアクセスしてしまうことがあります。

    例えば、以下のようなファイルが存在し、

      /sounds/audio.dat
      /sounds/japanese/audio.dat

    次のようなオーバーレイレイヤを設定します。

      addLayer("/", 0, 0, "/sounds", kDEFAULT, &layer1);
      addLayer("/", 0, 0, "/sounds/japanese", kDEFAULT, &layer2);

    このとき、アプリケーションが"/audio.dat"にアクセスした場合、layer2の 
    "/sounds/japanese/audio.dat" が選択されるべきですが、layer1の
    "/sounds/audio.dat"が選択されていました。

  - PSARC-MSELFアーカイブファイルに含まれるSDATAファイルを、ファイルの途中から
    読み込もうとすると、EFSSPECIFICエラーが返り読み込むことができません。
   （技術情報ID:200911-01 https://ps3.scedev.net/technotes/view/973）

- simple_logging、simple_mp3chain および simple_mp3stream サンプルの 
  makefile において、無効のコンパイラスイッチによる SNC ビルドの
  ビルド設定に関する警告が含まれていました不具合を修正しました。

- simple_create_fd において、NULL のカスタム定義が原因で NULL ポインタが
  再定義されている問題に関するビルド警告が含まれていました。これを削除し、
  NULL が正しくインクルードされるようにしました。

- SPU 上で動作するストリーム毎の OpenAL ユーザーロールオフカーブ
 （デフォルトで SPU 上で動作）が無視されていた不具合を修正しました。

- cellMSCoreSetMask() に CELL_MS_DSPALLSLOTS フラグを指定すると、
  メモリエラーが発生していました。この不具合を修正し、全スロットの
  マスクが設定されるようにしました。

- cellMSStreamClearRouting() が CELL_MS_ERROR_BUSMISMATCH でエラーに
  なっていた不具合を修正しました。
